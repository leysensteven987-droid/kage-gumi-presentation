壱What it is & why it exists

A WebCenter page is a stack of nested dashboards, embedded saved searches and task pages - and nothing on screen tells you which is which or where to edit them. The old reflex was to dig through the dashboard manager and guess. Tōshi reads the page you're already on and lays the structure bare.

見 Sees what's in front of you

Scoped to the page you opened it on - the dashboards/searches/task actually rendered, nested by their real DOM structure. Not a whole-instance crawl, just this page.

↗ Every name is a doorway

Each dashboard and search name links straight to its editor (dashboard.jsp?…&edit=1, the saved-search editor, …). See it, click it, edit it.

{} Sees the scripts too

One toggle and every dashboard script on the page shows up nested under its dashboard - display and run scripts, each with its type (visibility, on change, validation, …). The logic behind the glass, not just the layout.

⧉ Every window, at once

Open it in two browser windows, ten tabs, whatever - they never collide. Tōshi injects fresh on the page you're looking at and keeps no shared state, so there is nothing to clear, ever. The flaw that made the inspector this replaces unusable with more than one window open (clear the cache, every time) simply is not here.

弐Install (load unpacked)

Tōshi is a lean Chrome extension - our own code, no build step. It injects only when you click its icon (activeTab + scripting, no host permissions).

  1. Open chrome://extensions
  2. Turn on Developer mode (top-right toggle)
  3. Click Load unpacked and select the extension folder
    …\kage-gumi\extensions\toshi
  4. Pin 透 Tōshi to the toolbar (puzzle-piece icon → pin)
  5. Open a WebCenter page and click the 透 icon - the panel slides in. Click the icon again to toggle it off.

伍Two views - don't confuse them

Tōshi shows structure two different ways. Default is what's in front of you; the graph is the deep dependency map.

default

ON PAGE

The dashboards, searches and task page actually rendered on this page, nested by DOM containment - a parent is the nearest ancestor that's also inspectable. No network calls. Scoped to what you can see.

⤷ refs

REFERENCE GRAPH

Walks each on-page dashboard's full LOADDASHBOARD config - every dashboard it links to, recursively (sub / button / task / approval + embedded searches), cycle-safe, with DELETED dashboards flagged. The deep dependency map - opt in when you want it.

陸Good to know

  • It tracks the page. Highlights and the reveal-all outlines re-position as you scroll or resize.
  • Nested labels never hide each other. Reveal-all nudges colliding labels down so all stay readable.
  • The page stays usable. Only the labels are clickable - the outline boxes are click-through, so you can still work the page underneath.
  • Filter by id even with ids hidden. The match looks at the underlying text, not just what's on screen.
  • It reads your live session. The reference graph fetches same-origin, carrying your WebCenter cookie - nothing extra to log into.
  • It names the instance you're on. A bar under the header reads the live WebCenter version + host (from productinfo), so you never mistake which environment you're inspecting. Hidden on a non-WebCenter page.
  • Self-contained & safe. Renders in a shadow DOM (WebCenter's CSS can't leak in, and Tōshi never touches the page's own styles).
